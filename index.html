Drive the web browser

Use instructions to tell your web browser what to do. You can click links, fill out forms, press buttons and do anything a normal user would do. Watir also checks results, such as whether an expected text appears on the page.

Test the entire stack

Leave no stone unturned. By using a real web browser instead of stupid headless browsers you make sure you do not break anything for your users. Watir provides end-user emulation, ensuring that your entire stack (HTML, scripts, styling, embedded resources, backend functionality) is working.

What Is Watir?

Watir (pronounced water) is an acronym for “Web Application Testing in Ruby”, and is a toolkit used for automating interactions with web browsers. Watir allows you to write tests that are easy to read and maintain.

Combined with a behavior-driven testing framework such as RSpec, you can drive actual browsers to test your web applications, emulating user behavior and ensuring that your entire stack works well for end users.

Install and Use

To install OperaWatir:

jruby -S gem install operawatir

And, see all the commands:

operawatir --help

How we use watir at Opera

At Opera Software, we use Watir not only to test web applications, but also for testing the browser itself.

We have about 1,200 automated renderer tests running at each new build in our testing framework.
