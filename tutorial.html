Getting Started

The Watir API allows you to write scripts that interact with any web page. Its primary purpose is to ease test automation for web applications. Your scripts can, for example, before you deploy automatically go through all the steps your users normally would and alert you of any regressions.

In this tutorial we are going to control Google Maps. We will find directions from the Opera headquarters to the Oslo Opera House, and then we will take a closer look at our destination.

Setting up

First make sure you have installed OperaWatir. If you have trouble check that you have all the requirements [TODO], and if you still cannot get it working try contacting us on our mailing list.

% jruby -S gem install operawatir
% operawatir --help

The beginning

In every script you will first need to require OperaWatir, and then start Opera. Pretty soon after that you will want to visit a URL. This is simply achieved by setting the url property on the browser object. Finally we need to quit the browser.

require 'operawatir'

browser = OperaWatir::Browser.new
browser.url = 'http://maps.google.com'

browser.quit

Save this code into a file called tutorial.rb, and then you run it using this command:

% jruby tutorial.rb

Opera will start, wait a moment, and then go and visit Google Maps, and quit. Now that we can visit any page we want, it is time to interact with them!

Finding elements

You can find the ID of an element in Opera by right clicking it, selecting ‘Inspect Element’, and looking for the id="…" attribute in the window that appears. Try finding my ID!

To interact with the page you first need to find the element you want to use. You can then perform a variety of actions of which the most common are clicking and typing in text. Take a look at the API Documentation [TODO] for more.

In this example we are going to type in the Google Maps search field. There are two things you need to find an element with Watir: the type of element, and an identifying feature. The easiest way to uniquely identify an element is by using the ID, for the search field this is “q_d”. This is also a text box and so we use the text_field method.

search = browser.text_field(:id, 'q_d')

Check out the documentation [TODO] to see the other elements you can find.

Now that we have the search box we want to type something in it.

The keyboard

We can type text by using the set method, and press ‘special’ keys by using the key method. We will use these methods to search for the directions from Opera HQ to the Opera House, and submit the form by pressing the enter key. Including our progress so far this gives us the following code:

require 'operawatir'

browser = OperaWatir::Browser.new
browser.url = 'http://maps.google.com'

# Find the search box
search = browser.text_field(:id, 'q_d')

# Type text and submit the form
search.set 'Opera Software ASA, Oslo to Den Norske Opera & Ballett'
browser.key('Enter')

# Wait 3 seconds so that we can see the results
sleep 3

browser.quit

Try saving this code and running the script using the command in the above section. After loading Google Maps, Opera will wait a bit, and then start typing, and press enter. The sleep 3 will keep Opera open long enough for you to see that our search was ambiguous, and we need to click the first result.

The mouse

Clicking links is very simple and uses some of the techniques we have already learned. First we need to find the link to click, and then we can call the click method.

In our search the first suggested result is correct, which always has the ID of ddw_dll_1_0. Using the link method to search for only links, results in:

browser.link(:id, 'ddw_dll_1_0').click

Put this after the sleep 3 above.

Difficult elements

Sometimes the element you want does not have an ID. In these cases we need to use other tools. First check if the element has a unique title or name. Or, if you know it will always be in the same position you search by its index.

# Examples of alternative selectors
labs_image = browser.image(:title, 'Google Maps Labs')
search_button = browser.button(:name, 'btnG')
web_link = browser.link(:index, 2) # second link on the page

If you still cannot locate the element using these you can use XPath. XPath is can be very complicated, but we will be using only the simple selector //div[@log='pan_down']. This selects all the <div> elements whose log attribute equals 'pan_down'. In this case there is only one: the pan down button located at the top left of the map. 

pan_down = browser.element_by_xpath("//div[@log='pan_down']")
zoom_in = browser.element_by_xpath("//div[@title='Zoom In']")

To have a look at our destination we are going to pan down once, and then click the Zoom In button twice.

Final code

Here is the complete code. Because Google Maps is an AJAX application we have added sleeps to make sure things have loaded and been added to the DOM before we try and find them.

require 'operawatir'

browser = OperaWatir::Browser.new
browser.url = 'http://maps.google.com/'

puts 'Find the search box'
search = browser.text_field(:id, 'q_d')

puts 'Type text and submit the form'
search.set 'Opera Software ASA, Oslo to Den Norske Opera & Ballett'
browser.key('Enter')

sleep 1

puts 'Select the first result'
browser.link(:id, 'ddw_dll_1_0').click

sleep 2

puts 'Pan down'
pan_down = browser.element_by_xpath("//div[@log='pan_down']")
pan_down.click

puts 'Zoom in'
zoom_in = browser.element_by_xpath("//div[@title='Zoom In']")
zoom_in.click
zoom_in.click

puts 'Done'

sleep 10

# End
browser.quit

Conclusion

If you want something else to try, see if you can close the left panel (hint: it is an image with an ID of panelarrow2).

Hopefully this has been an enjoyable introduction to OperaWatir, and has shown you how easy it is to automatically control the browser. And you are not limited to what has been demonstrated here, have a look at the API Documentation [TODO] to see everything you can do.
